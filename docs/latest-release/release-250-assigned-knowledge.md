Current release version: **0.250.001**

Assigned Knowledge lets agent creators define the search scope an agent should use. When you select an assigned-knowledge agent in Chat, workspace search is enforced and the relevant scope controls become read-only.

## User Side

Agents can be bound to specific workspace sources, documents, and tags so they answer from the knowledge selected for their role.

## Admin Side

Availability may depend on tenant settings, governance policies, workspace scope, action configuration, or admin-managed identities. If you do not see this capability, contact your SimpleChat admin.

## Why It Matters

This matters because specialized agents can stay focused on the knowledge they were designed to use.

## How to Try It

1. Use assigned-knowledge agents when you need a purpose-built assistant for a known document set.
2. Review the knowledge context shown in Chat when an assigned-knowledge agent is selected.
3. Agent creators can configure workspace sources, documents, tags, and available actions during setup.
