Current release version: **0.250.001**

URL Access gives users a clearer way to bring web pages into a conversation while letting admins control safety policy, allowed domains, blocklists, page budgets, and source-review behavior.

## User Side

Users can paste URLs into Chat and have SimpleChat treat them as source links or plain text depending on the workflow and admin policy.

## Admin Side

Availability may depend on tenant settings, governance policies, workspace scope, action configuration, or admin-managed identities. If you do not see this capability, contact your SimpleChat admin.

## Why It Matters

This matters because links are a natural way to bring external context into a chat, but they need bounded, policy-aware handling.

## How to Try It

1. Paste a URL into Chat when you want SimpleChat to consider a source page.
2. Use plain text when you want to discuss a URL string without fetching it.
3. If a URL is blocked, it may be restricted by domain policy or safety controls.
