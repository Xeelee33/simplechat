Deep Research Setup Guide
Deep Research expands Web Search with bounded source-page review. It plans searches, reviews public source pages, records skipped URLs, and can save a research ledger artifact for the chat.
Deep Research fetches public web content from the app server. Keep the page, domain, timeout, and JavaScript rendering limits aligned with your organization's web access and compliance expectations.
Recommended Setup
  1. Configure and test Web Search via Foundry Agent first. Deep Research uses web-search citations and optional query planning as source seeds.
  2. Enable Deep Research for chat in this section.
  3. Decide whether every signed-in chat user can use Deep Research, or whether access should be restricted by an Enterprise App role.
  4. If restriction is required, assign users or groups to the DeepResearchUser app role before turning on the requirement.
  5. Save settings, then test with a prompt that asks a current question and enables Web Search or includes public URLs.
DeepResearchUser App Role Setup

Use this only when Require DeepResearchUser app role is enabled.

  1. Open the Azure Portal and go to Microsoft Entra ID.
  2. Open Enterprise applications and select the SimpleChat enterprise application.
  3. Open Users and groups, then choose Add user/group.
  4. Select the users or groups that should use Deep Research.
  5. Under role, select Deep Research User, which emits the DeepResearchUser role value.
  6. Complete the assignment and have affected users sign out and back in so their session includes the new role claim.
If the role is missing in an existing deployment, update the app registration roles from deployers/azurecli/appRegistrationRoles.json. Deployer version 1.0.4 includes the DeepResearchUser role definition.
Settings Reference
Setting What it controls Typical guidance
Enable Deep Research for chat Turns the feature on for eligible chat users. Enable after Web Search has been tested successfully.
Require DeepResearchUser app role Restricts use to sessions with the DeepResearchUser role claim. Use for tenant-scale governance. Leave off to allow all signed-in chat users.
Allow internal network hostnames Allows DNS hostnames that resolve to private/internal network addresses. Use only for approved internal sites. Literal IP URL targets, localhost, metadata hosts, link-local addresses, reserved addresses, and unspecified addresses remain blocked.
Activation Mode Deep Research starts only when the user selects it for the message. Web Search by itself remains a plain web-search flow and does not review source pages.
Max Pages per Turn Total reviewed source-page budget for one chat turn. Hard limit is 10. Lower it to reduce latency and outbound requests.
Max Seed Pages per Turn Limits the initial direct URL and web-search citation pages before child links are considered. Keep near the total page budget for broad review, or lower it to leave budget for linked pages.
Max User URLs per Turn Caps direct URLs accepted from the user's message. Extra URLs are recorded as omitted in the ledger.
Max Search Queries per Turn Caps query-planner variants sent to the configured Web Search agent. Hard limit is 8. Lower this if your search agent has tight request budgets.
Inspect linked source pages Allows Deep Research to follow selected policy-approved links from source pages. Leave enabled for deeper evidence coverage.
Plan multiple web search queries Lets the selected chat model propose current-message-only query variants. Useful for research prompts with multiple facets or named entities.
Save research ledger artifacts Saves a Markdown artifact listing queries, reviewed pages, skipped URLs, and coverage. Recommended for auditability and user-visible research traceability.
Use model-assisted source link planning Lets the selected chat model rank candidate links found on reviewed pages. Useful when source pages expose many archive, release, or pagination links.
Allow JavaScript rendering fallback Uses Playwright/Chromium when static HTML does not expose enough source content. Requires the browser runtime in the hosting environment. Disable if outbound rendering is not allowed.
Respect robots.txt Skips pages disallowed for the SimpleChat source-review user agent. Leave enabled unless your organization has a specific approved exception.
Allowed and Blocked Domains Restricts or excludes public domains before any page fetch is attempted. Use allowed domains for strict research scopes. Use blocked domains for known low-quality or disallowed sources.
Log Deep Research activity Writes activity summaries to application telemetry. Recommended for troubleshooting skipped pages, latency, and policy decisions.
Built-In Safety Boundaries
  • Only HTTP and HTTPS URLs are eligible for review.
  • Literal IP URL targets, localhost, metadata hosts, link-local addresses, reserved addresses, unspecified addresses, credentialed URLs, excessive redirects, and oversized pages are blocked.
  • DNS hostnames that resolve to private/internal addresses are blocked unless Allow internal network hostnames is enabled by an admin.
  • Unsupported content types are skipped instead of downloaded into chat context.
  • Every redirect target is revalidated before fetch.
  • Skipped pages are tracked in the ledger so admins and users can see coverage gaps.
Troubleshooting Checklist
  • If users cannot see Deep Research, confirm the global toggle is enabled and, if required, the user has the DeepResearchUser role in their current session.
  • If searches do not produce source pages, run the Web Search test button and verify the Foundry agent returns web citations.
  • If JavaScript-heavy pages are skipped, confirm Playwright Chromium support is available in the app runtime.
  • If pages are skipped unexpectedly, check domain policies, robots.txt behavior, timeout limits, and Application Insights activity logs.
